Basic Javascript etiqutte

Case sensitivity

Keywords

Javascript is a very case sensitive language. Dissimilar to HTML and CSS, one capital letter in the wrong place can cause errors. Keywords (var, for if etc) are case sensitive ways to identify various actions and they all must start in lowercase. Var, For and If's will not work. Named variables will carry their case sensitive names. If a variable or function has a capital, it must be supported through the entire process.

Camel Case

In Javascript, style does not matter but it is helpful in helpwing wit hthe code's readability and consistency. A easily visible and widely used method of writing Javascript is 'Camel Case'. The general rule is not to capitalise the first word and from then capitalise the first letter from the second word and beyond. For example: thisIsAnExampleOfCamelCase.

Styling

Spacing

'White space' has no affect on Javascript but it can be helpful for readability sake. Space can be added around functions, operators or value.
var carparts=["steeringwheel","wheel","window","door","fuel","engine","seatbelt"
var carparts = ["steeringwheel", "wheel", "window", "door", "fuel", "engine", "seatbelt"

Semicolons

Semicolons can be used to break up and define the end of a statement. But they are not always needed. For readability, it is probbaly best not to use them for small easy to understand statements. For longer statements it may be easier to use them and break things up a little and define ends to functions.

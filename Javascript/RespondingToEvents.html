Responding to events

Detcting the event

Events on a webpage are not created or casued since they happen all the time. It can be as simple as scrolling on a webpage or hovering your mouse over an image. An event listener can handle individual events and allow us to respond to them. There are 3 ways we can respond and the one chosen is ultimately up to the creator's preference.

Option 1 - Writing HS code into HTML

Simple but not widely recommended. It mixes up HTML and JS, it isn't portableand can be confusing.

Option 2 - Specify an element or event and write a function

The element is referenced by a variable and the event is chosen from a list of possible events. In this instance mouseover is the event for this section. Once the onmouseover event occurs, the function is called. In addition to this I have added onmouseout so it reverts to its usual style.

Option 3 - Use the addEventListener() function

This option specifies the parameter for the event we want to respond to (excludes the words on and off e.g. onmouseon used above) and at the end True or False. Various event listeners ahve been assigned to this section's title which is a benefit to this option.

The event's response

When the mouse is over this text, I will change text.

Clicking and holding will change the text. Letting go reverts it.

You may have received an alert saying "It has been more than five seconds". This is caused by setTimeout. 5 seconds after the page loads, this alert will appear

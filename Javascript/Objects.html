Objects

What is an object?

Learning objects are key to Javascript. If you understand objects, you understand Javascript.

Objects have properties and methods. All objects have properties but their values are probably different. Their methods may also be the same but they are performed at diferent times. An object contains these properties and are defined by them. The BEST way to think of objects is like a pizza

Properties

  • Toppings
  • Size
  • Cheese

Methods

  • Cooking
  • Preparing
  • Delivery

Objects can be: Booleans, Numbers, Stringnames, Dates, Maths, Expressions, Arrays and Functions. Primative data values can be objects but they need to be created as objects.

Creating objects

Objects can be created by using an object literal

or keyword new.

A 3rd way to create objects is by using an object constructor. The first two options only cretae single objects. This method allows the format to be re-used to create multiple objects of a single type.

These objects will contain the same properties & methods by contain different values.

Prototype

Every object has a prototype and that prototype is also an object. New objects inherit the prototypes Ps & Ms. Objects created and then edited will only edit their own Ps & Ms. A new object created from the prototype will not have the edited objects Ps & Ms. The prototype must be edited and changed for any new objects to share all of the Ps & Ms.

What's the difference to an array?

Arrays contain values in a list format. When we refer to an item within the array, we use 0 based indexing to retrieve the value. An object holds name:value pairs reffered to as properties.
array = [0, 1, 2, 3, 4, 5]
object = {fName: "Johnny", sName: "Bravo", eyeColour: "unknown", height: "6'3", bodytype: "muscular"}

Editing objects

Simply, a new property can be created and inserted into an object and an object can be removed by creating a new property and using the delete keyword.

This will only dit this object and NOT the prototype. All new objects created will be different.

We can also change our employees properties with methods:
let dailySchedule = 'Saturday'
let CEO = {
suit : 'Navy Blue Suit',
schedule: 'Busy meetings'
};

function outfit(obj){
obj['suit'] = 'Shorts & a t-shirt';
}

function workToday(obj){
if (dailySchedule === 'Saturday' || dailySchedule === 'Sunday'){
obj.workToday=false, obj.schedule="'Chillin'"
} else {
obj.workToday=true
}
}

console.log(CEO)
outfit(CEO)
workToday(CEO)
console.log(CEO)
The CEO is the object and has the value of a Navy Blue Suit and is scheduled for busy meetings. Below that are the functions where one will change the suit choice straight away and the other will only change the CEO's schedule value if the global variable dailySchedule is Saturday OR Sunday.

In the last block the CEO object will be shown by its default value, the functions will be called and the CEO's properties will have updated and displayed again.

Note: the dailySchedule is declared as a global variable since: 1, it might not only affect the CEO object and 2, it would be out of scope (undefinable) for the function.

Getters

const Employee = {
_startTime: 100,
get _startWork(){
if(typeof this._startTime === 'number'){
return 'My starting time is ' + this._startTime
} else {
return 'Unavailable data entry'
}
}
}
console.log(Employee._startWork)
Getters can perform an action on the data when getting a property. Getters can return different values using conditionals. In a getter, we can access the properties of the calling object using this. The functionality of our code is easier for other developers to understand.
